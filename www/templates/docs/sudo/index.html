sudo

sudo is a commonly used command on unix-based systems. Essentially, it allows a user to run a command as another user, by default the superuser. Originally, the command comes from the phrase ‘superuser do’ (sudo). The sudo system provides fine-grained control to clearly and simply define who can do what as superuser.

Password-less sudo is a convenient way to allow users to login to a machine with their own credentials and have access to root-required functionality. If you’d like to read a full background on this, take a look at the sudo.ws site.

To enable password-less sudo for a single user create a new file

/etc/sudoers.d/bigsql

Now, add the contents

bigsql      ALL=(ALL)     NOPASSWD: ALL

This essentially says, “the user bigsql can run all commands with no password”. Now, login as the bigsql user, you should be able to:

$ sudo id
uid=0(root) gid=0(root) groups=0(root) context=unconfined_u:unconfined_r:unconfined_t:s0-s0:c0.c1023

Notice, because I used ‘sudo’ the id command actually ran as the root user, so, root should be returned. If you get an error, try logging out of the ‘bigsql’ user and logging back in before running sudo.
