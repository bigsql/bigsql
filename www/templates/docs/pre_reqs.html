Pre-Requisites for BigSQL/Hadoop Developers

A reasonably powerful 64-bit developers workstation is required to be succesful with this bundle. You will be running a complete Hadoop and PostgreSQL stack of server components, all on one computer, in pseudo-distributed mode. You need to dedicate at least 6 GB of RAM to the server (or virtual machine) you configure.

Step 1) An up to date 64-bit Linux

CentOS/RHEL 6
su -c "yum upgrade"
Ubuntu 12.04 LTS
sudo apt-get update
sudo apt-get upgrade

 

Step 2) Make sure SSH Server is installed

RHEL
su -c "yum install openssh-server"
Ubuntu
sudo apt-get install openssh-server

After completing the above step, as a test, "ssh" to yourself using you're local (non-root) user account.

 

Step 3) Make sure local Firewall is disabled

RHEL
su -c "service iptables save"
su -c "chkconfig iptables off"
Ubuntu
sudo iptables -F
sudo ufw disable

After completing the above step, as a test, "ssh" to yourself using you're local (non-root) user account.

 

Step 4) Set the Hostname

RHEL Set the HOSTNAME to something like "mymachine.mydomain" in the file "/etc/sysconfig/network"
Ubuntu The file "etc/hostname" should have one line with "mymachine.mydomain"

After completing the above step, you need to re-boot so that this change takes effect. You should now be able to run the hostname command and have it echo back your hostname

 

Step 5) Set the Fully Qualified Domain Name (FQDN)

Linux Add a line to the end of your "/etc/hosts" file that identifies the fully qualified and short hostnames 
192.168.1.200   mymachine.mydomain   mymachine
Of course you will substitute in an appropriate ip address for your environment. With the above example names, the "hostname -s" command will display "mymachine" & the "hostname -f" command will display the FQDN as "mymachine.mydomain". You will be able to ping this machine by either the short or full name. Of course you will substitute in an appropriate ip address for your environment.

 

Step 6) Ensure SSH locally without a password (for local non-root account)

Linux 
ssh-keygen -t dsa -N "" -f "$HOME/.ssh/id_dsa"
touch $HOME/.ssh/authorized_keys
chmod 600 $HOME/.ssh/authorized_keys
cat $HOME/.ssh/id_dsa.pub >> $HOME/.ssh/authorized_keys

ssh mymachine.mydomain

Note: You will likely get a fingerprint warning the first time you ssh to your local machine.
