Upgrading PostgreSQL packages

The 2017-02-09 point releases of PostgreSQL by BigSQL introduced the powerful pgc command-line tool.

In order to upgrade from the non-pgc version of the PostgreSQL package to the latest, youâ€™ll need to follow a few simple instructions. Once you have upgraded to this new, latest release, future updates will be as simple as running 'pgc update pg96'.

Minor version upgrade ONLY

This document only applies to MINOR version upgrades:

  • 9.6.1 -> 9.6.2
  • 9.5.5 -> 9.5.6
  • 9.4.10 -> 9.4.11
  • 9.3.15 -> 9.3.16
  • 9.2.19 -> 9.2.20

If you are performing a major version upgrade:

  • 9.5.x -> 9.6.x
  • 9.4.x -> 9.5.x
  • etc...

This requires either a dump / restore or, the use of pg_upgrade. Major version upgrades are not covered by this document.

Stop PostgreSQL

Before continuing any further, stop the existing PostgreSQL installation. This can be done by executing
(as root):

service postgresql-96 stop

(Replace 9.6 with whichever version you have installed).

Backup your data

This is an optional, but highly recommended step.
The RPM uninstall we are about to perform will not destroy your existing database. However, it's always a good idea to make a backup (if possible) before proceding.
(as root)

cp -Rp /opt/postgresql/pg96/data /opt/data_save

Remove the previous RPM

yum erase postgresql96

Install the new RPM

Note: This procedure only applies to a minor version upgrade, make sure you are using the appropriate package.
(as root)

yum install -y http://oscg-downloads.s3.amazonaws.com/packages/postgresql-9.6.2-1-x64-bigsql.rpm

Enable autostart of the instance

The structure of the RPM has changed. In order for your old data directory to be autostarted, we will need to first initialize a new database in order to configure autostart. After initialization, we will remove the empty, new database and move your existing database into the proper location.

cd /opt/postgresql

./pgc init pg96

# (when prompted, provide the same superuser password as your previous instance)

./pgc config pg96 --autostart=on

Move old data in to new instance

cd /opt/postgresql/data/

mv pg96 pg96.old

mv /opt/postgresql/pg96/data /opt/postgresql/data/pg96

Start the database

(as root)

cd /opt/postgresql/

./pgc start pg96

At this point, your database should be up and running and your data will be available.
